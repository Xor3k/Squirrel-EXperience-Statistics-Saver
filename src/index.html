Подробнее
  1. Указывая не существующую директорию - она создаться сама.
  2. Так же если не удастся получить данные клана по каким-то причинам в ней же создаться что-то вроде лог файла, с возможными причинами ошибок и время попытки получить данные.
  3. Эти настройки сохраняются в файле settings.json. Где лежит этот файл было описано в описание, в readmi файле на github.

Подробнее
  1. Указывая ID нескольких кланов через запятую - будет сохраняться и запрашиваться сразу несколько кланов. Например: "144399, 144399". Сохраняться они будут в разных файлах. Лишние пробелы будут убраны.
  2. На каждый ID клана, на получения данных дается три попытки. Если все три попытки неудачны - создается лог ошибок. Где будет время получения неправильных данных и возможные причины.
    Ну а дальше - выполняется следующий запрос, если это не последний ID клана.
